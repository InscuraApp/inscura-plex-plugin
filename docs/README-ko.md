# Inscura의 Plex 메타데이터 플러그인

언어: [English](../README.md) | [简体中文](README-zh.md) | [日本語](README-ja.md) | **한국어**

Inscura는 영화 정보, 출연진, 장르, 컬렉션, 커버 및 배경 이미지 등의 자료를 정리할 수 있는 로컬 미디어 라이브러리 앱입니다. Plex는 미디어 라이브러리의 재생 및 관리를 담당하지만, Inscura에서 이미 정리해 둔 이러한 데이터에 대해서는 알지 못합니다.

이 플러그인은 Inscura의 로컬 API 서비스를 Plex에 연동하는 데 사용됩니다. Plex가 영화를 스캔할 때, 플러그인은 제목, 소개, 출시일, 평점, 장르, 태그, 컬렉션, 출연진, 감독, 각본가, 제작자, 커버 이미지 및 배경 이미지를 Plex 메타데이터에 기록합니다.

이 플러그인은 Inscura의 미디어 라이브러리 데이터와 라이브러리 내에서 생성된 이미지 리소스만 읽을 뿐, 원본 미디어 파일을 다운로드, 이동, 이름 변경, 삭제 또는 수정하지 않습니다.

## 현재 기능

- Plex에서 전달된 실제 파일 경로와 파일명을 우선적으로 매칭하여, Plex 제목이 수동으로 수정되어 발생하는 오매칭을 줄입니다.
- 경로 매칭이 불가능할 경우, 번호, 파일명 내 식별 코드 및 Plex 제목을 순차적으로 시도합니다.
- Plex에서 제공된 재생 시간을 보조 평가 기준으로 사용하여, 동일한 이름이나 유사한 파일의 매칭 정확도를 높입니다.
- 제목, 원제, 정렬 제목, 소개, 슬로건, 출시일, 연도, 평점, 콘텐츠 등급, 제작사, 장르, 국가, 태그, 컬렉션, 출연진, 감독, 각본가, 프로듀서, 포스터 및 배경 이미지를 포함하여 Plex가 수신할 수 있는 영화 메타데이터 필드에 데이터를 기록합니다.
- Plex 메타데이터 에이전트는 메타데이터 응답에서 예고편이나 메이킹 영상 항목을 생성할 수 없으므로, Inscura의 예고편 리소스는 Plex에서 지원하지 않는 필드에는 기록되지 않습니다.

## Inscura 로컬 API 서비스 활성화

1. Inscura를 실행하고 Plex와 동기화할 미디어 라이브러리를 엽니다.
2. 설정 메뉴의 API 서비스 설정으로 이동하여 로컬 API 서비스를 활성화합니다.
3. 인증 방식은 토큰 모드를 유지하는 것이 좋으며, 설정 페이지에 표시된 API 토큰을 저장해 둡니다.
4. Plex 서버가 설치된 기기에서 상태 확인 주소에 접속하여 서비스에 접속 가능한지 확인합니다.

예시:

```bash
curl "http://[ip]:28687/api/v1/health"
```

Plex와 Inscura가 같은 컴퓨터에 설치되어 있지 않은 경우, 플러그인의 서비스 주소란에 `127.0.0.1`을 입력해서는 안 되며, Inscura가 실행 중인 컴퓨터의 LAN 내 주소를 입력해야 합니다. 예:

```text
http://[ip]:28687
```

로컬 인터페이스 서비스는 현재 미디어 라이브러리의 수명 주기에 따라 실행됩니다. 미디어 라이브러리가 열려 있고 서비스가 활성화된 상태에서는 포트를 감시하며, **미디어 라이브러리가 잠기거나 닫히거나 애플리케이션이 종료되면 서비스가 중지됩니다.**

## 플러그인 파일

[GitHub](https://github.com/InscuraApp/inscura-plex-plugin/archive/refs/heads/main.zip) 또는 [릴리스](https://github.com/InscuraApp/inscura-plex-plugin/releases)에서 최신 플러그인 zip 파일을 다운로드하세요

이 저장소에는 Plex에 설치해야 하는 두 개의 파일이 포함되어 있습니다:


| 파일 | 설치 위치 | 용도 |
| -------------------- | ---------------------------------------------------------- | --------------------- |
| `Inscura.bundle`     | /PlexMediaServer/AppData/Plex Media Server/Plug-ins | 메타데이터 에이전트. 메타데이터 검색, 매칭 및 쓰기 담당   |
| `Inscura Scanner.py` | /PlexMediaServer/AppData/Plex Media Server/Scanners/Movies | 영화 파일을 스캔하고 실제 파일 경로를 매칭 프로세스에 전달 |


설치 후, Plex에서 다음을 확인할 수 있습니다:

- 스캐너: `Inscura Scanner`
- 에이전트: `Inscura`



## Plex 데이터 디렉터리

Plex의 실제 디렉터리는 패키지 출처, 시스템 버전, 볼륨 이름, 컨테이너 매핑 및 설치 방식에 따라 달라집니다. 아래는 일반적인 위치입니다. 기기의 설정이 이와 다를 경우, Plex 패키지 페이지, 컨테이너 매핑 또는 시스템의 실제 데이터 디렉터리를 기준으로 하십시오.

[Plex 플러그인 디렉터리를 찾는 방법?](https://support.plex.tv/articles/201106098-how-do-i-find-the-plug-ins-folder/)

Plex 데이터 디렉터리를 찾은 후, 이 플러그인은 다음 위치에 배치해야 합니다:

```text
Plex 데이터 디렉터리/Plug-ins/Inscura.bundle
Plex 데이터 디렉터리/Scanners/Movies/Inscura Scanner.py
```



## Plex에서 플러그인 활성화하기

1. Plex의 라이브러리 관리로 이동합니다.
2. 기존 영화 라이브러리를 편집합니다.
3. 고급 설정에서 스캐너를 `Inscura Scanner`로 변경합니다.
4. 에이전트를 `Inscura`로 변경합니다.
5. Inscura 서비스 주소와 API 토큰을 입력합니다.
6. 설정을 저장합니다.
7. 라이브러리를 다시 스캔합니다.
8. 기존 영화에 대해 다시 매칭하거나 메타데이터를 새로 고침하여, Plex가 Inscura의 매칭 식별자와 메타데이터를 기록하도록 합니다.



## 플러그인 설정 설명


| 설정 | 설명 |
| ------------- | ------------------------------------------------------------------- |
| Inscura 서비스 주소  | Inscura 로컬 인터페이스 서비스 주소. Plex와 Inscura가 같은 컴퓨터에 있지 않을 경우 반드시 LAN 주소를 입력해야 합니다 |
| Inscura 인터페이스 토큰  | 로컬 인터페이스 서비스가 토큰 인증을 사용하는 경우 입력합니다. Inscura가 인증 없음으로 설정된 경우 비워둘 수 있습니다 |
| 검색 결과 수 | 매칭 시마다 Inscura에서 요청하는 후보 수 |
| 자동 매칭 최저 점수 | 후보 점수가 이 값보다 낮을 경우, 플러그인이 해당 후보를 자동으로 채택하지 않습니다 |
| Inscura 컬렉션 가져오기 | 비활성화하면 플러그인이 Plex에 이미 존재하는 컬렉션을 전혀 수정하지 않습니다 |
| Plex 컬렉션 대체    | ‘컬렉션 가져오기’가 켜져 있을 때만 적용됩니다. 켜면 Plex의 기존 컬렉션을 모두 지우고 Inscura의 컬렉션 및 시리즈를 기록합니다; 끄면 Inscura의 컬렉션 및 시리즈만 추가합니다 |


컬렉션 설정 조합 규칙:


| Inscura 컬렉션 가져오기 | Plex 컬렉션 대체 | 결과 |
| ------------- | ---------- | --------------------------- |
| 비활성화 | 비활성화 또는 활성화 | 컬렉션에 기록하지 않고 Plex의 기존 컬렉션을 유지 |
| 활성화 | 비활성화 | Inscura 컬렉션 및 시리즈 추가 |
| 활성화 | 활성화 | Plex의 기존 컬렉션을 Inscura 컬렉션 및 시리즈로 대체 |




## 사용 권장 사항

- 처음 사용할 때는 소수의 영화를 선택하여 재매칭을 시도하고, 제목, 출연진, 커버 및 컬렉션이 예상대로 표시되는지 확인한 후, 데이터베이스를 일괄 갱신하는 것이 좋습니다.
- Plex에서 영화 제목을 수동으로 변경한 적이 있더라도, 플러그인은 제목을 유일한 기준으로 삼지 않고 실제 파일 경로와 파일명을 우선적으로 사용하여 매칭합니다.
- Inscura 서비스 주소나 토큰이 변경된 경우, Plex 플러그인 설정에서 이를 동기화하여 업데이트한 후 메타데이터를 새로 고쳐야 합니다.
- Inscura 미디어 라이브러리가 잠겨 있거나 닫혀 있는 경우, Plex는 메타데이터를 읽을 수 없습니다.



## 문제 해결



### Plex에서 Inscura 에이전트 또는 스캐너가 표시되지 않는 경우

1. `Inscura.bundle`이 `Plex 데이터 디렉터리/Plug-ins/Inscura.bundle`에 있는지 확인하십시오.
2. `Inscura.bundle/Contents`가 존재하는지 확인하십시오.
3. `Inscura Scanner.py`가 `Plex 데이터 디렉터리/Scanners/Movies/Inscura Scanner.py`에 있는지 확인하십시오.
4. Plex 프로세스가 해당 파일을 읽을 수 있는 권한이 있는지 확인하십시오.
5. Plex 미디어 서버를 재시작하십시오.
6. Plex 웹 버전을 다시 열고, 라이브러리 고급 설정으로 이동하여 확인하십시오.



### Plex에서 플러그인은 표시되지만 메타데이터가 없는 경우

1. Plex 서버가 설치된 기기에서 Inscura 상태 확인 주소에 접속합니다.
2. Inscura 미디어 라이브러리가 열려 있고, 로컬 인터페이스 서비스가 실행 중인지 확인합니다.
3. Plex 플러그인의 서비스 주소가 잘못된 `127.0.0.1`이 아닌지 확인합니다.
4. 인터페이스가 토큰 인증을 사용하는 경우, Plex 플러그인에 올바른 토큰이 입력되어 있는지 확인하십시오.
5. 영화에 대해 메타데이터 재매칭 또는 새로 고침을 실행하십시오.



### 커버 또는 배우 프로필 사진이 표시되지 않음

1. Plex 서버가 Inscura 서비스 주소에 접근할 수 있는지 확인하십시오.
2. 인터페이스가 토큰 인증을 사용하는 경우, 플러그인 토큰이 올바른지 확인하십시오.
3. Inscura에서 해당 미디어 또는 배우에 대해 사용 가능한 이미지 리소스가 실제로 있는지 확인하십시오.
4. 해당 영화의 메타데이터를 새로 고침하십시오.



## 플러그인 업그레이드

1. Plex 미디어 서버를 중지하거나 재시작할 준비를 하세요.
2. 기존 `Plex 데이터 디렉터리/Plug-ins/Inscura.bundle` 파일을 삭제하세요.
3. 새로운 `Inscura.bundle` 파일을 플러그인 디렉터리에 복사하세요.
4. `Plex 데이터 디렉터리/Scanners/Movies/Inscura Scanner.py` 파일을 덮어씁니다.
5. 권한을 수정합니다.
6. Plex 미디어 서버를 재시작합니다.

업그레이드 후에도 Plex 웹 버전에 이전 설정이 계속 표시된다면, 먼저 브라우저 페이지를 강제 새로고침한 다음 라이브러리 편집 창을 다시 열어주세요.
