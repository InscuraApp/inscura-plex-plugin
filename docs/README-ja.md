# Inscura の Plex メタデータプラグイン

言語： [English](../README.md) | [简体中文](README-zh.md) | **日本語** | [한국어](README-ko.md)

Inscura は、映画情報、キャスト、ジャンル、コレクション、カバー画像、背景画像などのデータを整理できるローカルメディアライブラリアプリです。Plex はメディアライブラリの再生と管理を担当しますが、Inscura で整理されたこれらのデータについては認識していません。

このプラグインは、InscuraのローカルAPIサービスをPlexに連携させるためのものです。Plexが作品をスキャンする際、プラグインはタイトル、あらすじ、公開日、評価、ジャンル、タグ、コレクション、キャスト、監督、脚本家、プロデューサー、カバー画像、背景画像をPlexのメタデータに書き込みます。

このプラグインは、Inscuraのメディアライブラリデータとライブラリ内で生成された画像リソースのみを読み取り、元のメディアファイルのダウンロード、移動、名前変更、削除、または変更は行いません。

## 現在の機能

- Plexから渡された実際のファイルパスとファイル名を優先して照合し、Plexのタイトルが手動で変更されたことによる誤マッチングを軽減します。
- パスによる照合が見つからない場合、番号、ファイル名内の識別コード、Plexのタイトル順に照合を試みます。
- Plexから渡された再生時間を評価の補助として使用し、同名または類似したファイルのマッチング精度を向上させます。
- Plexが受け付け可能な映画メタデータフィールドへの書き込みに対応しています。これには、タイトル、オリジナルタイトル、ソート用タイトル、概要、キャッチコピー、公開日、年、評価、コンテンツレーティング、制作会社、ジャンル、国、タグ、コレクション、キャスト、監督、脚本家、プロデューサー、ポスター、背景画像が含まれます。
- Plex メタデータエージェントは、メタデータレスポンスから予告編や特典映像のエントリを作成できないため、Inscura 内の予告編リソースは Plex がサポートしていないフィールドには書き込まれません。

## InscuraのローカルAPIサービスを有効にする

1. Inscuraを起動し、Plexと同期したいメディアライブラリを開きます。
2. 設定内のAPIサービス設定に移動し、ローカルAPIサービスを有効にします。
3. 認証方式はトークンモードのままにしておくことを推奨します。また、設定ページに表示されるAPIトークンを保存してください。
4. Plex サーバーが動作しているデバイスでヘルスチェックアドレスにアクセスし、サービスに接続できることを確認します。

例：

```bash
curl "http://[ip]:28687/api/v1/health"
```

PlexとInscuraが同じマシン上にない場合、プラグイン内のサービスアドレスには`127.0.0.1`を入力せず、Inscuraを実行しているコンピュータのLAN内でのアドレスを入力してください。例：

```text
http://[ip]:28687
```

ローカルインターフェースサービスは、現在のメディアライブラリのライフサイクルに合わせて動作します。メディアライブラリが開いており、サービスが有効になっている間はポートをリッスンしますが、**メディアライブラリがロックされたり、閉じられたり、アプリケーションが終了したりすると、サービスは停止します。**

## プラグインファイル

[GitHub](https://github.com/InscuraApp/inscura-plex-plugin/archive/refs/heads/main.zip) または [Releases](https://github.com/InscuraApp/inscura-plex-plugin/releases) から最新のプラグイン ZIP ファイルをダウンロードしてください

このリポジトリには、Plexにインストールする必要がある2つのファイルが含まれています：


| ファイル | 配置先 | 用途 |
| -------------------- | ---------------------------------------------------------- | --------------------- |
| `Inscura.bundle`     | /PlexMediaServer/AppData/Plex Media Server/Plug-ins | メタデータエージェント。メタデータの検索、照合、書き込みを担当   |
| `Inscura Scanner.py` | /PlexMediaServer/AppData/Plex Media Server/Scanners/Movies | 映画ファイルをスキャンし、実際のファイルパスを照合プロセスに渡す |


インストール後、Plex には以下が表示されるはずです：

- スキャナー：`Inscura Scanner`
- エージェント：`Inscura`



## Plex データディレクトリ

Plex の実際のディレクトリは、パッケージの入手元、システムバージョン、ボリューム名、コンテナマッピング、およびインストール方法によって異なります。以下に一般的な場所を記載しますが、お使いのデバイスと異なる場合は、Plex パッケージページ、コンテナマッピング、またはシステム内の実際のデータディレクトリを基準としてください。

[Plex プラグインディレクトリの探し方](https://support.plex.tv/articles/201106098-how-do-i-find-the-plug-ins-folder/)

Plex データディレクトリが見つかったら、本プラグインを以下の場所に配置してください：

```text
Plex データディレクトリ/Plug-ins/Inscura.bundle
Plex データディレクトリ/Scanners/Movies/Inscura Scanner.py
```



## Plex でプラグインを有効にする

1. Plex のライブラリ管理画面を開きます。
2. 既存の映画ライブラリを編集します。
3. [詳細設定] でスキャナーを `Inscura Scanner` に変更します。
4. エージェントを `Inscura` に変更します。
5. InscuraのサービスアドレスとAPIトークンを入力します。
6. 設定を保存します。
7. ライブラリの再スキャンを実行します。
8. 既存の映画に対して再マッチングまたはメタデータの更新を実行し、PlexがInscuraのマッチングIDとメタデータを書き込むようにします。



## プラグイン設定の説明


| 設定 | 説明 |
| ------------- | ------------------------------------------------------------------- |
| Inscura サービスアドレス  | Inscura のローカルインターフェースサービスアドレス。Plex と Inscura が同じマシンにない場合は、LAN アドレスを入力する必要があります |
| Inscura インターフェーストークン  | ローカルインターフェースサービスがトークン認証を使用する場合に入力します。Inscuraが認証なしに設定されている場合は、空欄のままにできます |
| 検索結果の数 | マッチングのたびにInscuraからリクエストする候補の数 |
| 自動マッチングの最低スコア | 候補のスコアがこの値を下回る場合、プラグインはその候補を自動的に採用しません |
| Inscura コレクションのインポート | オフにすると、プラグインは Plex 内の既存のコレクションを一切変更しません |
| Plex コレクションの置換    | 「コレクションのインポート」が有効な場合にのみ適用されます。有効にすると、Plex の既存のコレクションが削除され、Inscura のコレクションとシリーズが書き込まれます。無効にすると、Inscura のコレクションとシリーズが追加されるのみです |


コレクション設定の組み合わせルール：


| Inscura コレクションのインポート | Plex コレクションの置換 | 結果 |
| ------------- | ---------- | --------------------------- |
| オフ | オフまたはオン | コレクションには書き込まれず、Plexの既存のコレクションが保持されます |
| オン | オフ | Inscuraのコレクションとシリーズを追加 |
| オン | オン | Plexの既存のコレクションをInscuraのコレクションとシリーズで置き換え |




## 使用上のアドバイス

- 初めて使用する際は、まず少数の映画を選択して再マッチングを行い、タイトル、出演者、カバー画像、コレクションが期待どおりであることを確認してから、ライブラリを一括更新することをお勧めします。
- Plex内で映画のタイトルが手動で変更されていた場合でも、プラグインはタイトルを唯一の基準とせず、実際のファイルパスとファイル名によるマッチングを優先します。
- Inscuraのサービスアドレスやトークンが変更された場合は、Plexプラグインの設定で同期更新を行い、その後メタデータを更新する必要があります。
- Inscuraのメディアライブラリがロックされているか、停止されている場合、Plexはメタデータを読み取ることができません。



## トラブルシューティング



### Plex が Inscura エージェントまたはスキャナーを認識しない

1. `Inscura.bundle` が `Plex データディレクトリ/Plug-ins/Inscura.bundle` に存在することを確認してください。
2. `Inscura.bundle/Contents` が存在することを確認してください。
3. `Inscura Scanner.py` が `Plex データディレクトリ/Scanners/Movies/Inscura Scanner.py` にあることを確認してください。
4. Plex プロセスがこれらのファイルを読み取る権限を持っていることを確認してください。
5. Plex メディアサーバーを再起動してください。
6. Plex ウェブ版を再度開き、ライブラリの詳細設定で確認してください。



### Plex でプラグインは認識されるが、メタデータが表示されない場合

1. Plex サーバーが動作しているデバイスで、Inscura のヘルスチェック URL にアクセスします。
2. Inscura メディアライブラリが有効になっており、ローカルインターフェースサービスが起動していることを確認します。
3. Plex プラグイン内のサービスアドレスが誤った `127.0.0.1` になっていないことを確認します。
4. インターフェースがトークン認証を使用している場合は、Plexプラグインに正しいトークンが入力されていることを確認してください。
5. 動画に対して、再マッチングまたはメタデータの更新を実行してください。



### カバー画像やキャストのアバターが表示されない

1. Plex サーバーが Inscura のサービスアドレスにアクセスできることを確認してください。
2. インターフェースがトークン認証を使用している場合は、プラグインのトークンが正しいことを確認してください。
3. Inscura 内で、該当するメディアまたはキャストに利用可能な画像リソースが実際に存在することを確認してください。
4. その作品のメタデータを更新してください。



## プラグインのアップグレード

1. Plex メディアサーバーを停止するか、再起動の準備をします。
2. 古い `Plex データディレクトリ/Plug-ins/Inscura.bundle` を削除します。
3. 新しい `Inscura.bundle` をプラグインディレクトリにコピーします。
4. `Plex データディレクトリ/Scanners/Movies/Inscura Scanner.py` を上書きします。
5. 権限を修正します。
6. Plex メディアサーバーを再起動します。

アップグレード後も Plex のウェブ版に古い設定が表示される場合は、まずブラウザのページを強制リロードし、その後ライブラリ編集ウィンドウを再度開いてください。
